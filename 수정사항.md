# 코드 에디터 개선 사항 (2026-01-20)

요청하신 개선점을 바탕으로 `index.html`과 `script.js` 파일의 코드를 다음과 같이 수정했습니다.

## 1. UI 개선 (`index.html`)

### '엄격 모드(Strict Mode)' 체크박스 추가
- '개별 실행 모드' 체크박스 옆에 **'엄격 모드(Strict Mode)'**를 활성화할 수 있는 새로운 체크박스를 추가했습니다.
- 이를 통해 사용자는 자바스크립트의 `엄격 모드`를 손쉽게 켜고 끌 수 있습니다.

```html
<div class="checkbox-container">
  ...
  <label class="checkbox-label">
    <input type="checkbox" id="strict-mode-checkbox" name="strict-mode" />
    <span>
      <strong>엄격 모드</strong> (Strict Mode)
    </span>
  </label>
</div>
```

<br>

## 2. 기능 개선 (`script.js`)

### 1) 코드 입력창(Textarea)에서 Tab 키 동작 기능 추가
- 코드 입력창에 포커스가 있을 때 `Tab` 키를 누르면, 다음 UI 요소로 포커스가 이동하는 대신 **탭(tab) 문자가 입력**되도록 개선했습니다.
- 이를 통해 코드 작성 시 들여쓰기가 훨씬 편리해졌습니다.

### 2) 'Strict Mode' 실행 로직 구현
- '엄격 모드' 체크박스가 선택된 상태에서 코드를 실행하면, 코드 상단에 `'use strict';`가 자동으로 추가되어 엄격 모드 환경에서 코드가 실행됩니다.

### 3) 반환 값(Return Value) 및 콘솔 출력 형식 개선
- 객체, 함수 등의 반환 값을 더 명확하게 볼 수 있도록 `formatOutput` 헬퍼 함수를 구현하고 적용했습니다.

#### 주요 개선 내용
- **객체(Object) 출력**: 이전에는 `[object Object]`로 표시되던 객체들이 이제 **내용을 포함한 JSON 형식**으로 보기 좋게 출력됩니다.
  ```javascript
  // 예시: { a: 1, b: 2 }
  {
    "a": 1,
    "b": 2
  }
  ```
- **함수(Function) 출력**: 함수 선언문이나 함수를 반환하는 코드를 실행할 경우, **함수의 소스 코드**가 그대로 출력됩니다.
  ```javascript
  // 예시: function test() { return 1; }
  "function test() { return 1; }"
  ```
- `console.log` 출력에도 동일한 포맷팅을 적용하여, 객체나 함수를 콘솔에 찍었을 때 그 내용을 명확히 확인할 수 있습니다.

### 4) 히스토리 표시 개선
- 실행 히스토리에 현재 실행 모드(개별/통합)와 더불어 **'Strict Mode' 사용 여부**가 함께 기록되도록 수정했습니다.
  - 예: `통합 실행 모드 (Strict)`

## 추가 개선 사항 (2026-01-20)

### 함수 실행 결과 처리 로직 개선
- **인자 없는 함수 자동 실행**: 코드 실행 결과가 인자를 받지 않는 함수(zero-argument function)일 경우, 함수의 소스 코드를 출력하는 대신 **함수를 자동으로 실행하여 최종 반환 값을 출력**하도록 개선했습니다.
  - **이전 동작**: `() => 1 + 1` 실행 시, `() => 1 + 1` (함수 소스 코드) 출력
  - **개선된 동작**: `() => 1 + 1` 실행 시, `2` (함수 실행 결과) 출력
- **중첩 함수 처리**: `() => () => 5` 와 같이 함수를 반환하는 함수가 중첩된 경우에도, 최종 값에 도달할 때까지 재귀적으로 함수를 실행하여 결과를 출력합니다.
- **인자 있는 함수**: `(a, b) => a + b` 와 같이 인자가 필요한 함수는 기존과 동일하게 함수의 소스 코드를 출력하여 혼동을 방지합니다.
