# JS Runner 코드 문제점 분석

## 🔴 HTML 규칙 위반 사항

### 1. SEO 및 메타데이터 (필수)
- ❌ `<title>` 태그가 너무 일반적임
  - 현재: "JS Runner with History Mode"
  - 개선 필요: 더 구체적인 제목 필요

- ❌ `<meta name="description">` 태그 누락
  - 페이지 설명이 없어 SEO에 불리

- ❌ Open Graph 태그 누락
  - `og:title`, `og:description`, `og:image` 없음

### 2. 웹 접근성 (필수)
- ❌ Heading 구조 문제
  - `<h1>` 태그가 없고 `<h3>`부터 시작
  - 계층 구조가 올바르지 않음

- ❌ Form 접근성 문제
  - `<label>`이 `<textarea>`와 연결되지 않음
  - `for` 속성과 `id` 속성이 일치하지 않음

- ❌ Button type 누락
  - `<button type="submit">` 명시 필요

- ❌ Form action 비어있음
  - `<form action="">` → 올바른 URL 또는 `#` 필요

- ❌ `.sr-only` 스타일이 별도 reset.css 파일로 분리되지 않음
  - CSS 규칙에 따라 reset.css로 분리 필요

### 3. 시맨틱 마크업
- ❌ 시맨틱 태그 미사용
  - `<main>`, `<header>`, `<footer>` 대신 `<main>`, `<section>` 사용
  - 더 명확한 문서 구조 필요

- ❌ `<section>`에 heading 누락
  - 모든 section은 heading을 포함해야 함

### 4. 인라인 스타일 사용
- ❌ 다수의 인라인 `style` 속성 사용
  ```html
  <div style="margin-top: 10px; font-size: 0.9rem;">
  <button type="submit" style="margin-top: 10px; padding: 10px; cursor: pointer;">
  ```
  - 모든 스타일은 CSS 파일로 분리되어야 함

## 🔴 CSS 규칙 위반 사항

### 1. Reset CSS 누락
- ❌ Universal selector 리셋 없음
  ```css
  * { margin: 0; padding: 0; box-sizing: border-box; }
  ```

- ❌ `html { font-size: 10px; }` 설정 없음

- ❌ `body` 기본 설정 부족
  - `font-family` 시스템 폰트 스택 없음
  - `line-height: 1.5` (단위 없는 숫자) 설정 없음
  - `color: #333` 기본 색상 없음
  - `font-size: 1.6rem` 설정 없음

### 2. CSS 변수 미사용
- ❌ `:root`에 CSS 변수 정의 없음
  - 색상, 간격, 폰트 등을 변수로 관리하지 않음
  - 하드코딩된 값들 (예: `#f9f9f9`, `#007bff`, `10px`)

### 3. 단위 사용 규칙 위반
- ❌ **픽셀(px) 단위 남용** (가장 심각한 문제)
  - `padding: 2% 10%` → rem 단위 사용 필요
  - `font-size: 1rem` → 직접 px 계산이 아닌 rem 사용
  - `padding: 10px` → rem으로 변경
  - `min-height: 50px` → rem으로 변경
  - `margin-bottom: 15px` → rem으로 변경
  - `padding: 15px` → rem으로 변경

- ❌ **rem 단위 미사용**
  - font-size, padding, margin 등에 px 사용

- ⚠️ 허용되는 px 사용처는 지켜지고 있음
  - `border: 1px solid` ✅
  - `border-radius: 4px, 8px` ✅

### 4. 네이밍 규칙 위반
- ❌ **camelCase 사용** (kebab-case 사용 필요)
  - `.errorMessage` → `.error-message`
  - `.textarea` → `.text-area` (또는 더 의미있는 이름)

### 5. 코드 품질
- ❌ 인라인 스타일 사용
  ```html
  <div style="margin-top: 10px; font-size: 0.9rem;">
  ```

- ❌ 의미 없는 클래스명
  - `.content` → 너무 일반적
  - `.textarea` → form 요소 타입과 동일한 이름

### 6. 레이아웃 규칙
- ⚠️ Flexbox 사용은 적절함
- ❌ `gap` 속성을 사용할 수 있는 곳에서 margin 사용

### 7. 접근성
- ❌ Focus 스타일 정의 없음
  - `textarea:focus`, `button:focus` 등에 대한 스타일 필요

### 8. 반응형 디자인
- ❌ 모바일 퍼스트 접근법 미적용
- ❌ 미디어쿼리 없음
- ❌ 반응형 고려 부족

## 💡 개선 방향

1. **CSS 구조 개선**
   - reset.css 파일 생성
   - main.css에서 @import로 reset.css 불러오기
   - CSS 변수를 활용한 일관된 디자인 시스템

2. **HTML 구조 개선**
   - 시맨틱 태그로 명확한 문서 구조
   - 모든 접근성 요구사항 충족
   - SEO 최적화

3. **코드 품질 개선**
   - 모든 스타일을 CSS 파일로 분리
   - 의미있는 클래스명 사용
   - 일관된 네이밍 컨벤션

4. **사용자 경험 개선**
   - 키보드 접근성 강화
   - 명확한 focus 상태
   - 반응형 디자인 추가
