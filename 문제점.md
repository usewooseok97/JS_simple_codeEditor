### 해결된 문제 (2026년 1월 22일)

1.  **함수 실행 및 반환 값 문제 (`deepFunction` 예시)**
    *   **문제**: "통합 실행 모드"에서 여러 줄의 코드를 실행할 때, 마지막 줄이 표현식(`deepFunction()`)임에도 불구하고 명시적인 `return` 문이 없어 `undefined`가 반환되었습니다. 또한, 중첩 함수가 최종 값까지 모두 실행되어 사용자가 원하는 중간 단계의 함수를 확인할 수 없었습니다.
    *   **해결**:
        1.  "통합 실행 모드"의 실행 로직을 개선했습니다. 이제 코드를 분석하여 마지막 줄이 표현식인 경우, 해당 값을 반환하도록 동적으로 코드를 재구성하여 실행합니다. 이를 통해 REPL(콘솔)과 유사하게 마지막 코드의 실행 결과가 `Return Values`에 표시됩니다.
        2.  중첩 함수를 끝까지 실행하던 `resolveFunction`을 제거했습니다. 이제 `deepFunction()`을 호출하면 `() => () => 100`이 반환되어, 사용자가 원하는 만큼 `()`를 추가하여 실행 단계를 직접 제어할 수 있게 되었습니다.

2.  **객체 포매팅 문제 (`formatOutput`)**
    *   **문제**: `null` 값이 `[object Object]`로 표시되는 등 일부 값의 출력이 부정확했습니다.
    *   **해결**: `formatOutput` 함수를 수정하여 `null`과 `undefined`를 명확하게 문자열("null", "undefined")로 처리하도록 개선했습니다. 또한, `const data = { username: 'hojun', age: 10, height: 200, 배열 : [1,2,3] }; data`와 같은 객체 입력 시 JSON 형태로 올바르게 출력되는 것을 확인했습니다.

3.  **Strict Mode 실행 오류**
    *   **문제**: "개별 실행 모드"에서 Strict Mode를 사용할 경우, `'use strict'` 지시어가 잘못된 위치에 중복 추가되어 코드가 비정상적으로 동작했습니다.
    *   **해결**: 각 실행 모드("통합", "개별")에 맞게 `'use strict'`가 한 번만, 그리고 올바른 위치에 적용되도록 로직을 분리하고 수정했습니다.
