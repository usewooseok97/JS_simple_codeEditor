# plan0123.md: 2026년 1월 23일 변경사항

## 목적
코드 입력 및 히스토리 출력의 **사용자 경험(UX) 향상**을 위해 전문 코드 에디터 라이브러리 도입

## 변경 이유
1. 기존 `<textarea>`는 단순 텍스트 입력만 가능하여 개발자 도구 느낌이 부족했음
2. 코드 구문 강조(Syntax Highlighting)가 없어 코드 가독성이 낮았음
3. 히스토리 및 다운로드 파일에서 코드가 일반 텍스트로만 표시되어 가독성이 떨어졌음

---

## 변경사항 요약

### 1. CodeMirror 에디터 적용 (코드 입력창)
- **목적**: 코드 입력창을 전문 코드 에디터로 업그레이드
- **변경 내용**:
  - CodeMirror 5.65.13 라이브러리 CDN 추가
  - Dracula 테마 적용 (어두운 테마)
  - 줄 번호 표시, 자동 줄바꿈, Tab 크기 설정 등 에디터 옵션 적용
- **파일**: `index.html`, `script.js`

### 2. Prism.js 구문 강조 적용 (히스토리 출력)
- **목적**: 실행 히스토리의 코드에 구문 강조 적용
- **변경 내용**:
  - Prism.js 1.29.0 라이브러리 CDN 추가
  - VSCode Dark+ 테마 적용
  - 히스토리 아이템의 코드를 `Prism.highlight()`로 하이라이팅 처리
- **파일**: `index.html`, `script.js`

### 3. 다운로드 파일 형식 변경 (.txt → .html)
- **목적**: 다운로드한 히스토리 파일에서도 구문 강조 유지
- **변경 내용**:
  - 기존: 일반 텍스트 파일 (`.txt`)
  - 변경: HTML 파일 (`.html`)에 Prism.js 스타일 포함
  - 버튼 텍스트: "텍스트파일로 다운로드" → "html파일로 다운로드"
- **파일**: `index.html`, `script.js`

### 4. 기존 Tab 키 핸들러 제거
- **목적**: 코드 정리 (중복 기능 제거)
- **이유**: CodeMirror가 자체적으로 Tab 키를 처리하므로 기존 수동 구현 불필요
- **파일**: `script.js` (주석 처리)

### 5. CSS 수정
- **변경 내용**: placeholder 색상 스타일 추가
- **파일**: `main.css`
